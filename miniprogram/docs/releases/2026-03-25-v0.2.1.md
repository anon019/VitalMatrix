# v0.2.1 Release Notes

Release date: 2026-03-25

## Summary

This release focuses on stability, perceived performance, and user experience consistency across the mini program.

## Highlights

- Unified local-date generation to avoid wrong-day requests and trend mismatches.
- Added global login-state protection so expired tokens do not trigger repeated auto-login attempts.
- Reduced duplicate page loading by guarding first `onShow` refreshes and deduplicating concurrent `loadData` calls.
- Moved trends heart-rate recovery loading off the primary render path for faster first paint.
- Fixed nutrition summary meal counts and invalidated nutrition caches after upload/delete flows.
- Updated settings copy to reflect the current Polar authorization limitation more honestly.

## Versioning

- App version: `0.2.1`
- Recommended Git tag: `v0.2.1`

## Verification

- Ran `node -c` against all modified JavaScript files.
- Manual runtime validation in WeChat DevTools is still recommended for page lifecycle timing and backend response compatibility.
